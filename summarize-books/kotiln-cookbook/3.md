# 3. 코틀린 객체 지향 프로그래밍

## const와 val의 차이 이해하기

컴파일 타임 상수에 const 변경자를 사용한다. val 키워드는 변수에 한 번 할당되면 변경이 불가능함을 나타내지만 이러한 할당은 실행 시간에 일어난다.

## 사용자 정의 획득자와 설정자 생성하기

코틀린은 모든 것이 기본적으로 public이다. 따라서 데이터 은닉 원칙을 침해하는 것처럼 보인다. 코틀린 클래스에서 필드는 직접 선언할 수 없다.
